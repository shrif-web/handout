# تمرین Open Source

## مقدمه
هدف از این بخش، آشنایی شما با فرآیند مشارکت در توسعه‌ی پروژه‌های متن‌باز (open source) و یادگیری روند حل issue و ایجاد PR (pull request) است.
به طور خلاثه در این تمرین، کافی‌ست اعضای هر گروه مجموعا ۳ سوال در سایت Stackoverflow پاسخ دهند و ۶ issue در Github انجام دهند و لینک آن‌ها در یک فایل txt به همراه شماره دانشجویی اعضای گروه قرار دهند و در کوئرا آپلود کنند.

# Github
- **فعالیت مورد نظر:**
  - پیدا کردن و حل تعدادی issue تکنیکال در ریپازیتوری‌های عمومی مختلف در Github
  - ایجاد pull request متناظر برای هر ایشو حل شده
- **الزامات:**
  - هر گروه باید ۲ ایشو در هر یک از موضوع‌های backend، frontend و زیرساخت (در مجموع ۶ ایشو) حل کند.
  - ایشوهایی مانند اضافه کردن documentation یا پاسخ به سوالات کاربران پذیرفته نیست.
  - لازم نیست pull request لزوما merge شود.
- **منابع:**
    - آشنایی با کلیات open source contributing در Github:
      - [Getting Started with Contributing to Open-Source](https://stackoverflow.blog/2020/08/03/getting-started-with-contributing-to-open-source/)
     
	- نحوه‌ٔ ایجاد pull request برای یک ایشو:
      - [Linking a Pull Request to an Issue](https://docs.github.com/en/issues/tracking-your-work-with-issues/linking-a-pull-request-to-an-issue)

# Stackoverflow
- **فعالیت مورد نظر:**
  - پیدا کردن و پاسخ دادن به ۳ سوال کاربران در سایت Stackoverflow.
  - هر سوال باید مربوط به یکی از موضوعات backend، frontend یا زیرساخت باشد (از هر موضوع یک سوال). سوال انتخاب شده باید برچسب (tag) مربوط به موارد تدریس شده در درس را داشته باشد.
- **الزامات:**
  - پاسخ‌ها باید اصیل و نوشته شده توسط خود شما باشند.
- **منابع:**
  - آشنایی با نحوهٔ پاسخ‌دهی مناسب در Stackoverflow:
    - [Answer a Question on Stackoverflow](https://www.wikihow.com/Answer-a-Question-on-Stack-Overflow)

## نکات عمومی
- تمرین به صورت گروهی و در گروه‌های ۳ نفره انجام می‌شود.
- پاسخ‌ها باید به صورت لینک ارائه شوند و کافی‌ست همه‌ی لینک‌ها را در یک فایل txt قرار دهید و در کوئرا آپلود کنید. این فایل کافی‌ست شامل لینک‌های مربوطه باشد و برای هر لینک نام عضوی از تیم که مسئول پاسخ دادن یا pull request است، آورده شده باشد.
